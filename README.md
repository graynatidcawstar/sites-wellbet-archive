# sites-wellbet-archive

本仓库用于归档和发布多个独立 HTML 页面，内容以站点页面存档为主。仓库不针对任何特定域名或网站，仅作为静态页面存储与展示的通用空间。

## 项目简介

`sites-wellbet-archive` 是一个静态 HTML 页面归档仓库。通过 GitHub Pages 或其他静态托管服务，可以公开访问仓库中的页面文件。每个页面均为独立归档，不依赖后端服务，适合用于历史页面存档、临时页面发布或静态内容展示。

## 目录结构

```
├── index.html          # 仓库入口页面（可选）
├── pages/              # 归档页面存放目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有归档的 HTML 页面文件，按需命名。
- `assets/`：存放页面引用的公共资源文件，保持目录整洁。
- `index.html`（可选）：可作为仓库主页，列出归档页面索引。

## 页面归档说明

- 每个归档页面均为完整的 HTML 文件，包含独立的样式和脚本。
- 页面内容保持原始状态，不进行修改或优化，仅作存档用途。
- 所有页面均为静态内容，不涉及用户交互或数据收集。
- 归档文件命名建议使用英文小写、连字符分隔，便于 URL 访问。

## 维护说明

- 欢迎提交 Pull Request 添加新的归档页面，或修正现有页面中的明显问题（如链接失效）。
- 请勿添加包含恶意代码、侵权内容或违反平台政策的页面。
- 本仓库仅作技术归档用途，不提供任何形式的推广、背书或认证。
- 维护者会不定期清理无效或过期的归档文件。

## 使用方式

1. 克隆或下载本仓库。
2. 将需要归档的 HTML 文件放入 `pages/` 目录。
3. 如需引用公共资源，请放入 `assets/` 对应子目录。
4. 通过 GitHub Pages 或其他静态服务器部署即可访问。
